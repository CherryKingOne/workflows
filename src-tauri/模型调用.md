# 模型调用模块目录结构设计

## 概述

本模块采用 DDD（领域驱动设计）架构，支持三种模型类型：
- **Chat**: 文本对话模型
- **Image**: 图片生成模型
- **Video**: 视频生成模型

核心设计原则：
1. **插拔式架构**: 每个模型独立实现，低耦合，便于新增/删除
2. **注册表驱动**: JSON 格式注册表定义支持的模型
3. **YAML 配置**: 用户通过 `config.yaml` 配置要使用的模型
4. **类型隔离**: 不同类型模型分目录管理

### 1. 领域层 (Domain)

#### `domain/model/entities.rs`
```rust

    Chat,

    Video,

    pub id: ModelId,
    pub name: ModelName,
    pub model_type: ModelType,
    pub provider: String,
    pub is_available: bool,

    pub model_id: ModelId,
    pub api_key: ApiKey,
    pub endpoint: Option<Endpoint>,
    pub is_enabled: bool,

```

#### `domain/model/chat/entities.rs`
```rust

    pub base: Model,

    pub supports_streaming: bool,

/// Chat 请求
pub struct ChatRequest {
    pub model: ModelId,
    pub messages: Vec<Message>,
    pub temperature: Option<f32>,
    pub max_tokens: Option<u32>,

    pub content: String,

    pub usage: TokenUsage,
```

#### `domain/model/chat/repositories.rs`
```rust
/// Chat 模型调用器接口（领域抽象）
pub trait ChatModelCaller: Send + Sync {
    fn call(&self, request: ChatRequest) -> Result<ChatResponse, String>;
    fn stream(&self, request: ChatRequest) -> Result<ChatStream, String>;

```

### 2. 应用层 (Application)

#### `application/model/chat/services.rs`
```rust
/// Chat 模型服务 - 统一调用入口
pub struct ChatModelService {
    registry: Arc<ModelRegistryService>,
    callers: HashMap<String, Box<dyn ChatModelCaller>>,
}

impl ChatModelService {
    /// 根据 model_id 自动路由到对应的 caller
    pub fn chat(&self, request: ChatRequest) -> Result<ChatResponse, String> {
        let caller = self.resolve_caller(&request.model)?;
        caller.call(request)
    }
}
```

## 插拔式设计原则

### 核心原则：每个模型独立文件

```
infrastructure/model/chat/
├── mod.rs               # 仅做模块导出
└── deepseek_chat.rs     # 独立文件：类型 + 调用器
```

**为什么不用 provider 目录？**
- 不同模型参数可能完全不同
- 封装为公共代码会变成"万能接口"，后续难维护
- 独立文件 = 低耦合 = 易维护 = 易删除

### 新增模型步骤

1. **注册表添加模型定义**
   在 `models.json` 中添加：
   ```json
   "chat": {
    "deepseek_chat": {
      "name": "DeepSeek Chat",
      "description": "DeepSeek 通用对话模型，支持 64K 上下文，适用于日常对话、文本生成、代码编写等场景",
      "max_tokens": 64000,
      "supports_streaming": true
    }
  }
   ```

2. **配置文件启用模型**
   在 `config.yaml` 中配置：
   ```yaml
   models:
     chat:
       deepseek_chat:
         enabled: true
         api_key: "your-api-key"
   ```

3. **创建独立模型文件**
   创建 `infrastructure/model/chat/deepseek_chat.rs`：
   ```rust
   //! DeepSeek Chat 独立实现
   // 完整的请求/响应类型 + 调用逻辑
   // 不依赖其他模型代码
   ```

4. **在 mod.rs 中导出**
   ```rust
   pub mod deepseek_chat;
   // 只需要加这一行
   ```

### 删除模型步骤

1. 从 `models.json` 移除定义
2. 从 `config.yaml` 移除配置
3. 删除模型文件
4. 从 `mod.rs` 移除导出

**删除一个模型 = 删除一个文件**，不影响其他模型。

---

## 关键设计决策

1. **低耦合**: 每个模型独立文件，不共享调用逻辑
2. **高内聚**: 同一模型的类型定义和调用逻辑在同一文件
3. **开放封闭**: 新增模型不改现有代码，只添加新文件
4. **类型安全**: Rust 强类型保证配置正确性
5. **统一接口**: 领域层 `ChatModelCaller` trait 统一调用方式

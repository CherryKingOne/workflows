根据 `package.json` 的内容，我可以看到项目的脚本配置：

## 开发环境指令
- `npm run tauri:dev` - 开发模式下前后端联调，启动 Tauri 开发服务器

## 生产环境指令
- `npm run tauri:build` - 构建生产环境的 Tauri 应用程序

这个项目是一个基于 **Next.js + Tauri** 的桌面应用，Tauri 是一个用于构建桌面应用的框架。

### 完整的脚本列表：

| 脚本 | 命令 | 说明 |
|------|------|------|
| `dev` | `next dev` | 仅启动 Next.js 开发服务器（不包含 Tauri 后端） |
| `build` | `next build` | 构建 Next.js 前端 |
| `start` | `next start` | 启动 Next.js 生产服务器 |
| `lint` | `eslint` | 代码检查 |
| `tauri` | `tauri` | Tauri CLI 命令 |
| **`tauri:dev`** | `tauri dev` | **开发环境联调** |
| **`tauri:build`** | `tauri build` | **生产环境构建** |

所以生产环境的构建指令是：
```bash
npm run tauri:build
```

这会构建出可发布的桌面应用程序安装包（如 `.dmg`、`.app` 或 `.exe` 文件，具体取决于你的操作系统）。
